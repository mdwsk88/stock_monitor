Product Prototype

把群推送升级成会记忆的 A 股研究助理

这个原型展示下一阶段产品方向：企业微信仍然是入口，但消息不再只是“日报/晚报广播”， 而是可追问、可订阅、可反馈、可沉淀每日记忆的研究工作流。

PM Lens

产品目标不是多发消息，而是让用户少错过、少误判。

  • 01每条推送都解释“为什么现在值得看”。
  • 02用户能一键追问，把被动接收变成主动研究。
  • 03系统记住用户关注的股票和主题，减少无关噪音。
  • 04反馈会进入指标闭环，持续校准阈值和模板。

Message System

一屏研究卡片

固定消息结构，减少大段 AI 作文。用户先看到结论，再看到证据、风险和下一步可追问入口。

Template 建议优先

结论 -> 为什么现在 -> 证据 -> 反证风险 -> 继续追问

用统一模板承接早报、盘中、晚报和风险速递。每条消息必须有“为什么现在”，避免用户只看到一堆股票名。

Noise Control 少而精

只推三类消息

我的关注发生变化、全市场主线变化、硬风险出现。其余内容放到日报或用户追问里。

Follow-up

每条消息自带追问

让用户不用思考怎么问，直接点/复制“看同主线”“看反向风险”“为什么上榜”。

Trust

显式标出证据和推断

事实、聚合分、模型推断分开写，长期建立用户对机器人边界的信任。

Personalization

关注池与每日记忆

把已有主题观察池产品化。群用户可以关注股票或题材，系统按关注池生成更相关的推送。

Feedback Loop

从“我觉得不错”变成可度量

用户反馈不需要复杂，只要轻量。每次反馈都能变成阈值、模板和订阅策略的校准信号。
